Les données

Les données sont au coeur du Deep Learning ; c'est autour de leur apparition massive que le DL a pris vie. Faire du Deep Learning, c'est travailler avec des données ; les manipuler, les comprendre, les transformer, et enfin les interpréter. Pour aller plus loin en DL, il est utile de :

  • savoir ce qu'est une donnée : définition, provenance, génération, artefacts, ...
  • savoir manier des données (plus généralement, via les bibliothèques spécialisées comme numpy et pandas sous python)
  • savoir manier différents formats de données
  • savoir représenter des données de manière efficace (matplotlib au minimum)

Qu'est-ce qu'un algorithme ?

Un algorithme est une suite finie et non ambiguë d’opérations ou d'instructions permettant de résoudre un problème ou d'obtenir un résultat -Wikipedia

Un algorithme sera ici considéré comme un modèle, prenant des entrées et présentant un sortie.

Le Machine Learning

L'apprentissage automatique (en anglais machine learning, littéralement « l'apprentissage machine ») ou apprentissage statistique est un champ d'étude de l'intelligence artificielle qui se fonde sur des approches statistiques pour donner aux ordinateurs la capacité d' « apprendre » à partir de données, c'est-à-dire d'améliorer leurs performances à résoudre des tâches sans être explicitement programmés pour chacune -Wikipedia

Le Machine Learning

Le machine learning a créé un changement de paradigme dans la programmation classique ; on ne cherchait plus à trouver des réponses à partir de règles et de données définies, mais au contraire à trouver les règles permettant de passer des données aux réponses.

Les branches du machine learning

Il existe plusieurs branches dans le machine learning, qui définissent les métodes d'apprentissage et d'interaction avec les données :

  • Le supervised (ou semi-supervised) learning
  • L' unsupervised learning
  • Le reinforcement learning

Les limites du machine learning

Le Machine Learning n'est pas une boîte à miracles : il existe de nombreux problèmes auxquels ces algorithmes sont confrontés :

  • le manque de données / d'accès auxs données
  • les données biaisées
  • le manque de ressources
  • les mauvais choix (d'algorithmes, d'outils, d'utilisateurs)

Conclusions sur le Machine Learning :

  • C'est un des composants majeurs de l'IA
  • Permet l'apprentissage de tâches spécifiques

Les spécificités du Deep Learning

En quoi le Deep Learning se détache-t-il des autres méthodes du Machine Learning ?
L'arme du Deep Learning Le deep Learning utilise une méthode particulière du machine learning : le neurone (anciennement perceptron)

Les neurones sont des modèles issus du machine learning, inspirés des neurones humains (bien que loin de les égaler) La spécificité du Deep Learning est d'organiser ces neurones en plusieurs couches, créant ainsi une hiérarchie de modèles d'apprentissage. Un modèle en Deep Learning est donc une hiérarchie de modèles de machine learning !

Le modèle architectural du Deep Learning

En Deep Learning, une couche est donc un ensemble de modèles partageant des entrées communes. Ces couches se succèdent, on en distingue trois types :

- La couche d'entrée (Embedding Layer)

- La couche de sortie

- Les couches intermédiaires (Hidden Layer)

Le modèle architectural du Deep Learning

Les couches d'un modèle en Deep Learning sont en fait composées de neurones, d'où l'appellation "réseaux de neurones" !

Ces neurones sont les éléments qui permettent au réseau d'apprendre : chaque neurone est un modèle d'apprentissage. Ils prennent en entrée une ou plusieurs sorties des neurones des couches précédentes et leur sortie est envoyée vers les couches qui suivent. A l'exception des couches d'entrée et de sortie : elles sont particulières car n'interagissent pas qu'avec d'autres couches.

Le workflow en Deep Learning

En Deep Learning, le workflow est un peu plus épuré par rapport au machine learning. Le modèle va lui-même se charger de transformer les images et d'en extraire les features utiles.

Le modèle est donc bien plus indépendant que ses homologues en machine learning : il décide lui-même de l'information utile dans les données fournies ! C'est ce qui a poussé les premières publications à parler de 'machines autonomes'. Pourtant nous sommes encore loin des machines autonomes...

L'apprentissage

Comment les modèles apprennent-ils ? Quels processus sont à l'oeuvre lors de l'entrainement ?
L'apprentissage : oui mais où ?

Rappel : les couches d'un modèle sont composés de neurones, qui sont eux-même des modèles en machine learning !

Ainsi, ce sont ces neurones en réalité qui vont 'apprendre' ! Ces derniers apprendront à réagir de manière différente aux inputs qui lui seront fournis, en fonction des sorties des précédents neurones Mais comment ces neurones apprennent-ils la manière de réagir à chacun des inputs ?

Le neurone, késako ?

Les neurones sont des modèles d'apprentissage non linéaires qui se basent sur des poids - des variables internes, qui influent sur la sortie du neurone.

Ces poids sont modifiés au fur et à mesure de l'apprentissage pour que la sortie du neurone corresponde le plus souvent possible et le plus fidèlement possible aux sorties attendues

Les poids dans une couche

Rappel : les couches d'un modèle sont composés de neurones, qui sont eux-même composés de poids, ces poids variant selon les données rencontrées et influant sur les sorties du neurone concerné.

On peut donc ramener l'ensemble 'couches et neurones' aux poids de ces derniers quand on cherche à comprendre comment l'apprentissage a lieu dans une couche. Une couche est alors assimilable à l'ensemble des poids qui composent ses neurones.

L'entrainement, ou comment modifier les poids

Afin de déterminer si un modèle est compétent ou pas, il faut définir une métrique d'efficacité dudit modèle. Cette métrique est la Loss Function, ou fonction de perte. On peut alors pour chaque image, calculer un score par rapport à la sortie attendue

Ce score (loss score) est utilisé pour modifier les poids du réseau de neurones afin que la prochaine sortie ait un score moins élevé. L'outil qui transforme le score en une modification des poids est appelé Optimizer (optimiseur en français, du nom de sa fonction : modifier les poids pour optimiser la sortie du réseau )

Pourquoi tant de temps entre la théorie et les outils ?
Mais aussi ...

  • Améliorations des capacités de stockage
  • Améliorations des capacités en mémoire vive (cache)
  • Enorme avancées en captation, tri, partage des données

Or, ce sont justement les ingrédients qui empêchaient le deep learning de se développer : le manque de données, la difficulté de stocker de gros volumes de données, la capacité de calcul

Que fait réellement le Deep Learning ?

Que se passe-t-il réellement dans les couches d'un réseau de neurones ? Qu'est-il possible de faire ou pas avec un réseau ?
Caractéristiques du Deep Learning

  • Compositions de multiples transformations non linéaires
  • Construction de plusieurs niveaux d'asbtraction qui aident à donner un sens aux données
  • Construction hiérarchique des caractéristiques et représentations

Le Deep Learning permet de modifier la représentation des données initiales, pour les présenter sous une forme qui fera apparaître la solution au problèmep posé au réseau

Les couches d'un modèle

Les couches d'un modèle pour la vision par ordinateur s'organisent ainsi :

  • Basses couches : patterns simples (traits, coins)
  • Couches intermédiaires : patterns modérés (formes, contours )
  • Couches profondes : patterns complexes (motifs,parties d'objets )

L'organisation en couches permet une représentation hiérarchique des patterns à reconnaitre. Ces patterns, de plus en plus complexes, sont utilisés pour reconnaitre les éléments à l'image. Ce sont ces patterns qui sont appris par les neurones

Ce que fait ou non le Deep Learning

On a vu que les modèles en Deep Learning étaient capables d' identifier des patterns simples comme complexes. Ceci est valable en traitement d'image, mais aussi de texte, d'audio, etc... A partir de là, il est bon de rappeler les limitations du Deep Learning :

  • Pas de prévision du futur
  • Pas ou peu de capacité face à des données nouvelles
  • Pas de performance sur une tâche trop générale
Si les modèles développés récemment arrivent à exceller dans certains domaines complexes, ils restent limités par les contraintes énoncées ci-dessus.

Remarque : Ces limitations ne sont pas physiques ou matérielles, mais conceptuelles, internes au concept même du Deep learning tel qu'il existe aujourd'hui.

Conclusion

Le Deep Learning est un outil puissant, qui a révolutionné le monde de l'intelligence artificielle depuis sa percée dans les années 2010.

Cet outil est soumis à des limitations qui empêchent pour le moment le développement d'Intelligence Artificielles telles qu'on les voit souvent présentées dans les récits futuristes.

Pour autant, dans bien des domaines spécialisés, des algorithmes de DL ont réussi à égaler voir surpasser l'humain, principalement sur des problèmes complexes.

Pourtant, les tâches 'simples' réalisées par les enfants en bas âge sont encore innateignables pour ces algorithmes.
